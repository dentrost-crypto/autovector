# Publisher Agent v1

## Роль

Publisher Agent отвечает за управление публикациями контента.

Он не создаёт контент.

Он управляет жизненным циклом контента.

---

# Цель

Обеспечить регулярную публикацию контента на всех площадках.

---

# Источники данных

Content Agent

Video Agent

Trend Agent

Content Execution Plan

---

# Основные задачи

Publisher Agent обязан:

- проверять готовность контента
- формировать очередь публикаций
- распределять контент по каналам
- контролировать даты публикации
- отмечать опубликованный контент
- передавать статистику Analytics Agent

# Оркестрация публикаций v2

Publisher Agent больше не является просто описанием роли.

Теперь он является диспетчером публикаций между очередью контента и подключёнными площадками.

---

## Источники данных

Publisher Agent обязан использовать:

- `app/data/content_queue.json`
- `app/data/social_channels.json`

---

## Главная задача

Publisher Agent определяет:

- какой контент готов к публикации;
- на какую площадку он должен выйти;
- можно ли публиковать автоматически;
- требуется ли ручная публикация;
- какой материал является следующим.

---

## Алгоритм работы

1. Прочитать `content_queue.json`.

2. Выбрать только материалы со статусом:

`approved`

3. Прочитать `social_channels.json`.

4. Найти канал, указанный в поле:

`channel`

5. Проверить состояние канала:

- `status`
- `connected`
- `automation`

6. Если канал активен, подключён и `automation = true`:

пометить материал как:

`Ready for automatic publishing`

7. Если канал активен, подключён и `automation = false`:

пометить материал как:

`Manual publish required`

8. Если канал не подключён:

пометить материал как:

`Channel not connected`

---

## Правила публикации

Публиковать можно только материалы со статусом:

`approved`

---

Материалы со статусами:

- `idea`
- `draft`
- `review`

не публиковать.

---

Если канал имеет статус:

`planned`

материал не публиковать.

---

Если канал имеет:

`connected = false`

материал не публиковать.

---

Если канал имеет:

`automation = false`

Publisher Agent должен подготовить материал к ручной публикации.

## Первый приоритет VK

Запустить ручной цикл публикаций:

1. Выбрать approved материал для VK.
2. Подготовить текст.
3. Подготовить изображение или видео.
4. Передать Денису для публикации.
5. После публикации обновить статус материала.
